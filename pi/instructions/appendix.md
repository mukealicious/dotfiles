## Pi Appendix

- Use `exa_search` first for ordinary public-web discovery, coding docs, API examples, and quick current lookups.
- Use `parallel_search` only when Exa is unsuitable or unavailable; use `parallel_research` for broad deep research/synthesis; use `parallel_extract`/`parallel_enrich` for Parallel-specific extraction and enrichment.
- Use `bash`/`curl` for known URLs, APIs, raw files, localhost, and downloads.
- Keep paid-search calls bounded: start with 1-3 targeted searches and fetch only high-value sources.
